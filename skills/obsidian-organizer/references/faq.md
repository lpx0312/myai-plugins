# 常见问题 (FAQ)

## 标签相关

**Q: 标签用中文还是英文？**
A: 技术术语优先英文（K8S、Docker、Jenkins），其他可中文（安装配置、故障排查）

**Q: category 和 tags 区别？**
A: category 是主分类（单值，用于目录），tags 是多维度标签（用于检索）

**Q: 如何避免标签过多？**
A: 每个笔记 3-7 个标签，定期合并重复标签

**Q: 为什么我的标签太笼统？**
A: 旧版本的 tag_taxonomy.md 使用 `工具/CentOS` 这种形式，太正式。新版本使用混合维度标签：
  - 技术栈：`Linux`、`K8S`、`Docker`
  - 具体操作：`ens33转eth0`、`Pod重启`、`磁盘分区`
  - 场景：`安装配置`、`故障排查`、`监控告警`

**Q: 如何确保标签可搜索？**
A: 标签名就是你平时搜的关键词：
  - 搜 "ens33" → 找到 `ens33转eth0` 标签
  - 搜 "网卡" → 找到 `网卡配置` 标签
  - 搜 "GRUB" → 找到 `GRUB配置` 标签

## 格式相关

**Q: wiki 链接怎么写？**
A: `[[文档名]]`，如果标题有空格用 `[[]]` 包裹

**Q: status 字段可以用 emoji 吗？**
A: ✅ 可以！`status: ✅ 已验证` 是正确的。但标签中不要用，应该是 `tags: [- 状态/已验证]` 而非 `tags: [- 状态/✅ 已验证]`

**Q: 旧笔记需要全部改吗？**
A: 不需要。新笔记严格按规范，旧笔记按需整理常用即可

**Q: 顶部的 `#标签` 要改吗？**
A: 必须迁移到元数据 `tags`，并**删除正文中的旧标签行**：
   - 默认：迁移标签到元数据 + 删除正文中的 `#标签` 行
   - 仅优化元数据：迁移标签到元数据 + 保留正文（需明确指定）
   - 这样可以统一使用 Obsidian 的标签管理功能

**Q: 如何识别旧格式 `#标签`？**
A: 识别规则：
   - ✅ 识别：单独一行的 `#nacos安装`、`#k8s部署`（# 后无空格）
   - ❌ 不识别：标题格式 `# 一级标题`（# 后有空格）
   - ❌ 不识别：代码块中的 `#`（可能是注释）

**Q: 什么时候应该使用表格？什么时候不应该？**
A:
  - **适合表格**：参数说明、配置项、命令参考、对比分析、属性列表、错误码
  - **不适合表格**：纯叙述内容、代码示例为主、层级嵌套关系、内容项少于 2 个
  - **判断标准**：如果列表项有多个属性/字段可以提取，就转表格；如果只是简单列举，保留列表

## 文档关联

**Q: 相关文档和参考资源有什么区别？**
A:
  - **相关文档**：链接到笔记库中**实际存在**的笔记，使用 `[[笔记名]]` 格式
  - **参考资源**：链接到外部网页，优先官方文档，使用 `[标题](URL)` 格式
  - 只添加真实存在、有实际关联的链接

**Q: 如何避免添加不存在的 wiki 链接？**
A:
  - 整理笔记前先浏览笔记库，了解已有哪些笔记
  - 只链接到确实存在的笔记
  - 如果相关的笔记还不存在，可以添加到"待办"列表，稍后创建
  - 或者补充外部官方文档链接作为参考

**Q: 应该添加多少个相关文档？**
A:
  - 推荐 2-5 个相关文档
  - 优先链接最直接相关的笔记
  - 避免过度链接导致混乱
  - 质量大于数量

## 元数据字段

**Q: 元数据中的 `sources` 字段和正文中的"参考资源"有什么区别？**
A: 这是一个常见混淆点！

**元数据 `sources` 字段**（在 `---` 之间）：
```yaml
---
sources:
  - https://blog.example.com/article
  - https://docs.example.com/api
---
```
- ✅ 只支持纯 URL 格式
- ❌ **不支持** Markdown 链接格式 `[标题](URL)`
- 用于记录参考来源的 URL

**正文"参考资源"**（在 `---` 之后）：
```markdown
## 参考资源

- [技术博客标题](https://blog.example.com/article) - 补充说明
- [官方文档](https://docs.example.com/api)
```
- ✅ 支持 Markdown 链接格式
- ✅ 可以添加标题和说明
- 用于展示可点击的链接

**如果元数据中使用 Markdown 链接格式会怎样？**
- Obsidian 无法正确渲染
- YAML 解析器可能报错
- 链接无法点击

**记住**：元数据用纯 URL，正文用 Markdown 链接

## 图片相关

**Q: 整理笔记时需要把外链图片下载到本地吗？**
A: 不一定。**保持稳定外链优先，仅在需要时本地化。**

| 场景 | 处理方式 |
|:-----|:---------|
| 自己的 CDN（如 `tc.sktill.top`）| **保留外链**，命名已足够清晰 |
| 第三方稳定图床（Imgur、Github 等） | **保留外链** |
| 临时 URL（含随机 token、时间戳参数） | 下载本地化 |
| 404 或无法访问的图片 | 下载或删除 |
| 用户明确要求离线可用 | 下载本地化 |

**Q: 下载图片时如何命名？**
A: 优先使用**原文件名**，不要改成 `downloaded_xxx.png`。例如原图 `20210113124543.png` 保持不变，或根据内容语义重命名为 `01-Yum仓库架构图.png`（须语义明确）。
