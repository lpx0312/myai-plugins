# Obsidian 格式规范速查

**重要**：详细格式规范（wikilinks、callouts、代码块、表格等）请参考官方 [obsidian-markdown skill](obsidian-markdown)。

本文档仅包含 obsidian-organizer 特有的格式约定。

## 元数据 (Frontmatter)

### 最小配置
```yaml
---
tags:
  - 工具/Docker
category: 容器技术
---
```

### 推荐配置
```yaml
---
created: 2026-01-23
updated: 2026-01-23
tags:
  - 工具/Docker
category: 容器技术
---
```

### 可选字段
```yaml
difficulty: 初级/中级/高级
importance: 低/中/高
status: ✅ 已验证/🚧 草稿/🔄 进行中/❌ 已废弃
sources:
  - https://example.com
```

### 字段说明

#### tags
- **类型**：列表，用 `-` 表示
- **数量**：建议 3-7 个标签
- **层级**：使用 `/` 分隔，最多 2-3 层
- **⚠️ 重要**：标签中不要使用 emoji

#### category
- **类型**：单一值
- **用途**：表示主分类，通常对应目录名称

#### status
- **类型**：枚举值
- **⚠️ 特殊**：status 字段可以使用 emoji

```yaml
# ✅ status 字段：可以使用 emoji
status: ✅ 已验证

# ✅ tags 字段：不要用 emoji
tags:
  - 状态/已验证
  - 类型/故障排查
```

## 标签与 status 的区别

```yaml
# ✅ status 字段：可以使用 emoji
status: ✅ 已验证

# ✅ tags 字段：不要用 emoji
tags:
  - 状态/已验证
```

## 文档结构

```markdown
# 标题（可选加状态 emoji，如 ✅ 已验证）

> 简短描述

## 📖 一、概述/背景

## 🧩 二、核心概念

## 🔧 三、常用方法

## 🧪 四、实战示例

## 📌 五、注意事项

## 总结

- 要点1
- 要点2

## 相关笔记

- [[笔记1]]
- [[笔记2]]

## 参考资源

- [链接](https://example.com)
```

**标题层级**：
- `#` H1：文档标题（仅1个）
- `##` H2：主要章节
- `###` H3：子主题
- `####` H4：细节说明

**标题 Emoji 标记规范（推荐使用）**

为章节标题添加 emoji 前缀，提升可读性和视觉层次感：

| Emoji | 含义 | 适用场景 |
|:------|:-----|:---------|
| `📖` | 概述/介绍 | 概念说明、背景介绍 |
| `🧩` | 基础语法/要素 | 语法结构、组件说明 |
| `🔧` | 常用工具/方法 | 工具命令、方法函数 |
| `⚙️` | 配置/设置 | 配置项、参数说明 |
| `🧠` | 原理/分析 | 核心概念、原理讲解 |
| `🪄` | 技巧/诀窍 | 实用技巧、黑科技 |
| `🧼` | 格式/清理 | 格式化、空格处理 |
| `🧪` | 示例/实战 | 示例代码、实战演练 |
| `🧾` | 模板/清单 | 模板片段、命令清单 |
| `📌` | 注意事项 | 关键点、常见问题 |
| `✅` | 验证/完成 | 验证步骤、完成状态 |
| `❌` | 错误/警告 | 错误示例、避坑指南 |
| `💡` | 提示/建议 | 小技巧、经验分享 |
| `🚧` | 进行中/待定 | 未完成、待补充 |

**参考示例（kubectl go-template 笔记）**：

```markdown
# ✅ kubectl `-o go-template` 全量语法速查表

## 🧩 一、Go-template 标准语法结构

## 🔧 二、常用函数与操作符

## 🧠 三、kubectl 不支持的函数（⚠️ 常见误区）

## 🪄 四、无 `add` 场景下模拟计算技巧

## 🧼 五、空格与换行控制技巧表

## 🧪 六、实战场景模板范例

## 🧾 七、整合输出模板（标准写法）

## 📌 八、调试与建议
```

**使用原则**：
- ✅ H2 章节标题建议添加 emoji 前缀
- ✅ emoji + 中文数字序号 + 章节名：`## 🧩 一、核心概念`
- ❌ 避免过度使用，一个标题不要超过 1 个 emoji
- ❌ 不要在 H1 标题前加 emoji（H1 通常已有状态 emoji）

## 文件命名

### 推荐格式
- **技术文档**：`工具名称-用途.md` → `Docker-基本使用.md`
- **故障排查**：`问题简述-故障排查.md` → `Pod频繁重启-故障排查.md`
- **SOP**：`流程名称-SOP.md` → `生产环境发布-SOP.md`
- **知识整理**：`主题名称-整理.md` → `微服务架构-设计原则.md`

### 命名原则
- ✅ 清晰表达内容
- ✅ 避免特殊字符
- ✅ 简洁但有意义
- ❌ 避免过长
- ❌ 避免无意义
