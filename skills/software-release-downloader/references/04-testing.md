# 阶段 4: 测试验证

## 验证步骤

### 1. 添加可执行权限

```bash
chmod +x {name}-downloader.sh download.sh
```

### 2. Dry-run 模式测试

先用 dry-run 模式测试，预览将下载的文件：

```bash
timeout 60 ./download.sh -n
```

根据打印日志判断：
- 如果全部或者大部分显示 `[状态: 可用]` 则说明测试通过
- 只要 dry-run 测试通过，就说明脚本可以交付

### 3. 实际下载测试 (可选)

如果用户有测试机器，建议先在测试环境验证：

```bash
# 实际下载测试
./download.sh
```

## 常见问题排查

### API 限流

如果看到 `403 Rate limit exceeded` 错误：
- 确保设置了有效的 `GITHUB_TOKEN`
- 减少请求频率或等待一段时间

### URL 不可用

如果某些版本显示 `[状态: 不可用]`：
- 检查该版本是否确实存在
- 某些预发布版本会被过滤掉

### 代理连接失败

如果下载失败：
- 检查代理服务器是否可用
- 确认代理地址和端口正确
